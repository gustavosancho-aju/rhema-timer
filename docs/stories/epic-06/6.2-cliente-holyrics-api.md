## Contexto

Para enviar comandos ao Holyrics (exibir versículo), precisamos de um cliente HTTP que fale com a API local do Holyrics 2.28+ na porta 8091. O token de autenticação é configurado pelo usuário uma vez.

## User Story

**Como** sistema RHEMA
**Quero** enviar comandos HTTP autenticados para o Holyrics local
**Para** exibir versículos na projeção do culto em < 2 segundos.

## Especificação técnica validada

### Endpoint base
```
http://localhost:8091/api/{action}?token={TOKEN}
```
- **Método:** POST
- **Content-Type:** application/json
- **Token:** query parameter `?token=...`

### Actions que vamos usar

| Action | Propósito | Payload |
|---|---|---|
| `ShowVerse` | Exibir versículo | `{ "references": "Is 43:19" }` |
| `CloseCurrentPresentation` | Ocultar texto atual | `{}` |
| `GetBibleVersionsV2` | Listar traduções (opcional) | `{}` |

### Formato de resposta
```json
// Sucesso
{ "status": "ok", "data": {} }

// Erro
{ "status": "error", "error": "invalid token" }
```

## Validação manual antes de codar

Faça esse curl na máquina do Holyrics (substituindo o token) pra confirmar:

```bash
curl -X POST "http://localhost:8091/api/ShowVerse?token=SEU_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"references":"Is 43:19"}'
```

Resposta esperada: `{"status":"ok","data":{}}`

**Se retornar erro de parsing da referência em português, tentar:**
- `"references": "Isaías 43:19"` (nome completo)
- `"id": "23043019"` (formato BBCCCVVV)
