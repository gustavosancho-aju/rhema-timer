# Story 7.5 — Interface Viewer

**Epic:** 07 — Timer de Apresentação
**Status:** Ready for Review
**Prioridade:** Alta
**Depende de:** Story 7.3
**Estimativa:** 3 pontos

---

## Contexto

O Viewer é a tela que o apresentador vê no palco (projetada em monitor de confiança). Deve ser minimalista, legível à distância, ocupar a tela inteira e atualizar em tempo real sem nenhuma interação do usuário.

---

## Critérios de Aceitação

- [ ] Rota `/timer/[roomId]/viewer` renderiza em tela cheia (fullscreen auto)
- [ ] Fundo preto por padrão
- [ ] Tempo exibido em fonte grande (ocupa ~60% da tela)
- [ ] Título do timer exibido abaixo do tempo (fonte menor)
- [ ] Nome do apresentador exibido (opcional, abaixo do título)
- [ ] Barra de progresso horizontal na parte inferior:
  - Verde quando > 50% do tempo restante
  - Amarelo quando < 50%
  - Vermelho quando < 20% (wrap-up)
- [ ] Mensagem recebida do Controller aparece em destaque sobre o timer
- [ ] Estado de conexão: indicador sutil (ponto verde/vermelho no canto)
- [ ] Se `blackout` ativo: tela completamente preta
- [ ] Se `flash` ativo: tela pisca brevemente
- [ ] Se `focus_mode` ativo: só mostra mensagem, esconde timer
- [ ] Timer continua rodando localmente mesmo sem WebSocket (Web Worker)
- [ ] Ao reconectar, sincroniza estado automaticamente
- [ ] Suporta modo `time_of_day`: exibe hora atual do dispositivo
