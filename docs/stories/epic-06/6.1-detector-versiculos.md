# Story 6.1 — Detector Bíblico (Endpoint + Prompt)

**Epic:** EPIC-6 — Integração Holyrics + Detector de Versículos
**Status:** Done ✅ (validado em 23/04/2026)
**Prioridade:** Alta
**Estimativa:** 2h
**Depende de:** nada (pode rodar em paralelo com EPIC-5)

## Contexto

Precisamos de um agente IA especializado em detectar versículos bíblicos na fala da pastora — tanto citações diretas ("Isaías 43:19") quanto alusões ("eis que farei uma coisa nova"). Esse é o cérebro do sistema de sugestão ao Holyrics.

## User Story

**Como** sistema RHEMA
**Quero** analisar a transcrição em streaming e detectar versículos citados ou aludidos
**Para** gerar sugestões de exibição no Holyrics com referência + texto + confiança.

## Acceptance Criteria

- [ ] Novo endpoint `POST /api/detectar-versiculo` criado
- [ ] Endpoint recebe `{ transcricao: string }` (janela de texto dos últimos ~10-20s)
- [ ] Endpoint retorna JSON estruturado:
  ```json
  {
    "detectou": true,
    "livro": "Isaías",
    "capitulo": 43,
    "versiculo_inicio": 19,
    "versiculo_fim": 19,
    "texto_sugerido": "Eis que farei uma coisa nova...",
    "confianca": 0.85,
    "tipo": "citacao_direta" | "alusao"
  }
  ```
- [ ] Se não detectar, retorna `{ detectou: false }`
- [ ] Usa Claude Haiku 4.5 (mais barato, suficiente para esta tarefa)
- [ ] Threshold mínimo de confiança: 0.7 (abaixo disso retorna `detectou: false`)
- [ ] System prompt do agente em `src/lib/prompts/detector-biblico.ts` (versionado e commitado)

## Tarefas Técnicas

- [ ] Criar `src/lib/prompts/detector-biblico.ts` com system prompt especializado
  - Lista dos livros da bíblia em pt-BR
  - Instruções para identificar citações diretas e alusões
  - Formato JSON estruturado de saída
  - Exemplos few-shot de detecção correta e falso positivo
- [ ] Criar `src/app/api/detectar-versiculo/route.ts`
- [ ] Usar `query()` do `@anthropic-ai/claude-agent-sdk` com model Haiku
- [ ] Parsear JSON do output
- [ ] Validar estrutura e retornar 400 se input inválido (transcrição vazia)
- [ ] Testes unitários com fixtures de transcrições reais:
  - Citação direta com referência completa
  - Alusão sem referência
  - Fala que NÃO é versículo (negativo)
  - Múltiplos versículos na mesma janela

## Riscos e mitigações

- **Alucinação de versículos**: usar temperatura baixa (0.2) e threshold alto (0.7)
- **Falso positivo em linguagem bíblica genérica**: prompt deve diferenciar "linguagem espiritual" de "citação específica"
- **Custo**: Haiku a $1/1M tokens input, rodando a cada 20s = ~R$ 1,50 por culto (documentado no business case)
