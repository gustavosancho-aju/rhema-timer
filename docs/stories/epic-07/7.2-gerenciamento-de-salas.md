# Story 7.2 — Gerenciamento de Salas

**Epic:** 07 — Timer de Apresentação
**Status:** Ready for Review
**Prioridade:** Alta
**Depende de:** Story 7.1
**Estimativa:** 2 pontos

---

## Contexto

O usuário precisa criar e gerenciar salas. Cada sala gera links únicos para cada interface (controller, viewer, operator, moderator, agenda). O modelo é: uma sala = um evento/apresentação.

---

## Critérios de Aceitação

- [ ] Página `/timer` lista todas as salas existentes
- [ ] Botão "Nova Sala" abre modal para inserir nome
- [ ] Sala criada persiste no SQLite com ID nanoid (ex: `abc123`)
- [ ] Ao clicar em uma sala, exibe painel com os 5 links:
  - `/timer/[roomId]/controller`
  - `/timer/[roomId]/viewer`
  - `/timer/[roomId]/operator`
  - `/timer/[roomId]/moderator`
  - `/timer/[roomId]/agenda`
- [ ] Cada link tem botão "Copiar" ao lado
- [ ] Sala pode ser renomeada (inline edit)
- [ ] Sala pode ser deletada (com confirmação)
- [ ] API REST funcional:
  - `GET /api/timer/rooms` → lista todas
  - `POST /api/timer/rooms` → cria nova
  - `PATCH /api/timer/rooms/[id]` → renomeia
  - `DELETE /api/timer/rooms/[id]` → deleta
