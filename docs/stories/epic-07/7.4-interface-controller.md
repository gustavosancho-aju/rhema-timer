# Story 7.4 — Interface Controller

**Epic:** 07 — Timer de Apresentação
**Status:** Ready for Review
**Prioridade:** Alta
**Depende de:** Stories 7.2, 7.3
**Estimativa:** 5 pontos

---

## Contexto

O Controller é a interface principal de controle. O operador usa essa tela para gerenciar todos os timers da sala: play, pause, stop, reset, criar novos timers, editar durações e enviar mensagens ao apresentador. É a interface mais complexa do sistema.

---

## Critérios de Aceitação

- [ ] Rota `/timer/[roomId]/controller` renderiza a interface completa
- [ ] Header mostra: nome da sala + contador de conexões ativas
- [ ] Lista de timers da sala exibida em ordem
- [ ] Cada timer na lista mostra:
  - Título + apresentador (se houver)
  - Duração / tipo
  - Status (idle, running, paused, finished) com indicador visual
  - Controles: ▶ Play / ⏸ Pause / ⏹ Stop / ↺ Reset
  - Botão de editar (modal inline)
- [ ] Timer ativo destacado visualmente com borda colorida
- [ ] Botão "+ Adicionar Timer" abre formulário:
  - Título (text)
  - Apresentador (text, opcional)
  - Tipo (countdown / countup / time_of_day)
  - Duração (mm:ss, só para countdown)
  - Cor do label (12 opções)
- [ ] Timer pode ser reordenado via drag-and-drop (ou botões ↑↓)
- [ ] Timer pode ser deletado
- [ ] Painel de mensagens à direita (implementado na Story 7.7, stub aqui)
- [ ] Indicador de conexões ativas (número de viewers conectados)
- [ ] Ao pressionar Play, emite WebSocket `timer:start` para toda a sala
- [ ] Ao pressionar Pause/Stop/Reset, emite evento correspondente
- [ ] Controles de display (Blackout, Flash) — stub (implementados na Story 7.10)
