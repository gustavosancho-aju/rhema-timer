# Story 6.3 — UI de Cards de Sugestão

**Epic:** EPIC-6 — Integração Holyrics + Detector de Versículos
**Status:** Implementado ✅ (aguardando teste ponta a ponta)
**Prioridade:** Alta
**Estimativa:** 3h
**Depende de:** Stories 6.1 e 6.2

## Contexto

Quando o detector (6.1) identifica um versículo, precisamos mostrar uma sugestão visual ao operador com opção de 1-clique para exibir no Holyrics ou descartar. O design precisa ser rápido de escanear durante o culto ao vivo.

## User Story

**Como** operador de mídia durante o culto
**Quero** ver cards de versículos sugeridos aparecendo em tempo real
**Para** poder aprovar ou descartar a exibição com 1 clique, sem atrasar a projeção.

## Acceptance Criteria

- [ ] Nova seção "Versículos Detectados" aparece ao lado (ou abaixo) da transcrição ao vivo
- [ ] Cada card exibe:
  - Referência (ex: "Isaías 43:19")
  - Texto do versículo sugerido (primeiras ~100 chars)
  - Confiança do detector (badge colorido: >0.85 verde / 0.7-0.85 amarelo)
  - Tipo: "Citação" ou "Alusão"
  - Timestamp (ex: "há 3 segundos")
- [ ] Cada card tem 2 botões grandes:
  - **"Exibir no Holyrics"** (primário, destaque)
  - **"Descartar"** (secundário)
- [ ] Clicar "Exibir" chama o cliente Holyrics (6.2) e:
  - Mostra loading spinner no card
  - Em caso de sucesso: card fica verde e marca "Exibindo"
  - Em caso de erro: card fica vermelho com mensagem curta
- [ ] Clicar "Descartar" remove o card com fade-out
- [ ] Animação suave quando novo card aparece (slide-in from top)
- [ ] Limite visual: mostrar últimos 5 cards, scroll nos mais antigos
- [ ] Card "ativo" (atualmente exibindo) fica fixo no topo com botão "Ocultar no Holyrics"

## Tarefas Técnicas

- [ ] Criar componente `<VersiculoCard>` em `src/components/versiculo-card.tsx`
- [ ] Criar componente `<VersiculosSugeridos>` em `src/components/versiculos-sugeridos.tsx`
- [ ] Estado local (React state ou Zustand leve) para lista de sugestões
- [ ] Worker/hook que:
  - A cada 20s pega os últimos 20s de transcrição
  - Chama `/api/detectar-versiculo`
  - Se `detectou: true`, adiciona card à lista
  - Dedupe: se mesma referência em janela de 1 min, não duplica
- [ ] Integrar com `holyricsClient.exibirVersiculo()`
- [ ] Integrar no layout principal (side-by-side com transcrição, ou abaixo em mobile)
- [ ] Estilos Tailwind: dark-mode compatível, legível em condições de culto (lights dimmed)

## Decisões de UX

- **Sem auto-exibição** (decisão já tomada no business case — sempre aprovação humana)
- **Debounce 3s** no detector pra evitar múltiplas detecções do mesmo trecho
- **Som sutil** (opcional, toggle) quando nova sugestão aparece
- **Atalhos de teclado:** `Enter` aprova card ativo, `Esc` descarta

## Definição de Pronto

- Componentes criados, testados, integrados
- Animações suaves e responsivas
- Testado ponta a ponta: falar versículo → card aparece → clicar → Holyrics recebe
- Lint + typecheck passando
- Screenshots/gif no PR
